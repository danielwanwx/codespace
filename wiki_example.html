aggregate_edges

Resolve unqualified function calls to qualified directed edges across the entire repository.

Graph Resolution Function graph_aggregator.py:35

Overview

aggregate_edges is the core graph-building function in Codespace. It takes the flat list of extracted symbols (functions, classes, methods) and the raw file contents, then resolves each symbol's unqualified call targets into fully-qualified directed edges.

This is the most critical step in the pipeline because source code only contains unqualified names (e.g., scan_repo), but the graph needs qualified edges (e.g., codespace::indexer::scan_repo). The resolution uses a 4-priority system that leverages import statements, module scope, and global uniqueness to disambiguate call targets.

The output is two edge lists: function-level edges (for the detailed graph) and module-level edges (for the high-level view). Both are consumed by build_codespace_graph() during export.

How It Works

  1. Build import maps — For each file, calls parse_imports() to create a mapping of imported names to their source modules.
  2. Build symbol index — Creates a lookup table of all symbols by their short name (last component of qualified name), grouped by module.
  3. Resolve each call — For every symbol's calls list, invokes _resolve() which applies the 4-priority resolution:
    • Priority 1: Name appears in the file's import map → resolve to imported module's symbol
    • Priority 2: Name exists in the same module → resolve to same-module symbol
    • Priority 3: Name is globally unique across the repo → resolve to that symbol
    • Priority 4: Ambiguous → skip (filtered out)
  4. Aggregate edges — Counts duplicate edges between the same source/target pair, producing weighted edges.
  5. Generate module edges — Collapses function-level edges to module-level by extracting the module component from each qualified name.

Usage Example

# After extracting symbols and collecting file contents
all_symbols = []
file_contents = {}
for mod in modules:
    for f in mod.files:
        symbols = extract_symbols(f.content, repo_name, mod.slug, f.rel_path)
        all_symbols.extend(symbols)
        file_contents[f.rel_path] = f.content

build_reverse_index(all_symbols)

# Resolve the call graph
func_edges, mod_edges = aggregate_edges(all_symbols, file_contents)
# func_edges: [ResolvedEdge(source="codespace::cli::main", target="codespace::indexer::scan_repo", weight=3), ...]
# mod_edges:  [ResolvedEdge(source="codespace::cli", target="codespace::indexer", weight=5), ...]

Architecture Context

Edge Cases & Notes

Common names filtered: Builtins like print, len, range, str, int, list, dict are excluded via the COMMON_NAMES set to avoid noise in the graph.
Ambiguous calls dropped: When a call name matches multiple symbols across different modules and isn't imported, the edge is silently dropped rather than guessing. This is a deliberate design choice favoring precision over recall.
Self-edges excluded: A symbol calling itself (recursion) does not produce an edge.
